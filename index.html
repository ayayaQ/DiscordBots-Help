Getting Your Own Bot

1. Go here and login.

2. Once on the dashboard click the "New Application" button.

3. Name the application whatever you want.

4. In the app dashboard press the menu button and select "Bot"

5. Press "Add Bot".

6. Now you can copy your token and change other aspects of the bot account.

Inviting The Bot

Inviting the bot is also done from here!

Click on the OAuth2 button from the menu

In scopes check "bot"

Scroll down and select your bot permissions, "Administrator" counts for all permissions if you need them all."

Scroll back up to the scopes and copy then open the link. Note: You must have the "Manage Server" permission in the server you wish to add the bot to.

Creating Commands

First, navigate to the commands section and press the floating action button in the bottom right. This opens a menu to create your commands

You can use the dropdown menu to select which type you want for your command.

The command field is used for what will trigger the command.
NOTE 1: * in the command field will mean anything will trigger the command.
NOTE 2: For custom emojis in the reaction add type you will use the name of the emoji (eg. your emote is named WeirdChamp, you would then put WeirdChamp), and regular emojis will just use the emoji ðŸ˜Š in the command field.

Phrase means what is in the command field can appear anywhere in a message to trigger a command. Eg. Command is "hello" with phrase checked, if someone says "Hello there!" the command will trigger.

The command description field is optional and can be used to remind you what a command does.

Channel message will send a message in the channel where the command is triggered. You can use keywords in this field.

Private message will send a private message to someone who triggers a command. You can also use keywords in this field.

The moderation section (kick, ban, voice mute) works in a special way. Usage is whatever your command is with a mention directly after it. For example if you had kick checked and named your command "remove" you would call it will "remove @user" to kick the user.

Assign/Remove Role can assign and remove roles. The role name field is case sensitive and must match the name of the roles you want to assign. When this command is called the first time, it adds the roles to the user who called it. On the second time, it removes the role from the user who called it.

Keywords

Keywords are used to create more dynamic messages for your commands. They go into the output fields of your command like channel message or private message. Here is a list of the available ones. NOTE: * is an exception. It is a wildcard keyword for the command field. This means any message or reaction can trigger your command.

$name (Compatible with all except member leave and member ban) - This mentions the user who sent the message. NOTE: If you use it in the channel send part of an event where the user leaves you will receive an invalid user mention.

$channel (Compatible with Message Received) - Adds the channel name to the message.

$server (Not compatible with Private Message Received) - Adds the server name to the message

$message (Compatible with Message Received and Private Message Received) - This adds the message content of the message sent.

$rollnum(min,max) (Compatible with all) - This generates a random number where you replace min with the smallest number and max with the largest one. For example, if you made a dice command that uses $rollnum(1,6) it would show replace where you put the $rollnum(1,6) with the result. If your command was to reply with "$rollnum(1,6) is the roll of your dice." it would appear as below.

But if you made a mistake in your command and put in a number larger than the max in the min spot like "$rollnum(55,12)" You could get an error that appears like so.

$date (Compatible with all) - The current date.

$avatar (Compatible with all except member leave and member ban) - The current avatar of the user who called the command.

$defaultAvatar (Compatible with all except member leave and member ban) - The default avatar of the user.

$memberCount (Not compatible with pm recieved) - Gives the number of members currently in the server.

$serverCreateTime (Not compatible with pm recieved) - Gives the time the server was created.

$messageAfterCommand (Compatible with Message Received and Private Message Received) - gives all raw message content after command.

$commandCount - gives the number of commands bot has.

$namePlain - name of user who called command without mention.

$serverIcon (Not compatible with pm recieved) - icon of the server

$serverBanner (Not compatible with pm recieved) - banner of the server, if it exists

$serverDescription (Not compatible with pm recieved) - description of the server, if it exists

$serverSplash (Not compatible with pm recieved) - splash of the server, if it exists

$botAvatar - avatar of the bot

$botName - name of the bot

$mentionedAvatar (Only message recieved) - avatar of mentioned user

$mentionedName (Only message recieved) - name of mentioned user in a mention

$mentionedNamePlain (Only message recieved) - name of a mentioned user in plain format

$random{} - Used to select a random message or link for embed images. Used like $random{abc|def|ghi} where | is the seperator between choices to be picked.

$discriminator - discriminator of command caller

$tag - tag of command caller

$timeCreated - account creation time of command caller

$ID - id of command caller

$serversSharedWithBot - number of servers shared with command caller

$channelID (Only message recieved or react add) - id of channel where message is

$channelCreateDate (Only message recieved or react add) - date of the creation of the channel

$channelAsMention (Only message recieved or react add) - mention of channel

$botDiscriminator - discriminator of the bot

$botTag - tag of bot

$botDefaultAvatar - default avatar of bot

$botTimeCreated - creation time of bot

$botID - id of bot

$ping - ping to gateway

$inviteURL - url to invite the bot

$serverCount - amount of servers bot is in

$allMemberCount - amount of users in all servers the bot is in

$mentionedDiscriminator (Only message recieved) - discriminator of mentioned user

$mentionedTag (Only message recieved) - tag of mentioned user

$mentionedIsBot (Only message recieved) - is the mentioned user a bot?

$mentionedDefaultAvatar (Only message recieved) - default avatar of mentioned user

$mentionedTimeCreated (Only message recieved) - account creation date of mentioned user

$mentionedID (Only message recieved) - id of mentioned user

$mentionedServersSharedWithBot (Only message recieved) - how many servers the mentioned user shares with the bot.

$eval (version 1.7+) Used to mark the start of a block of javascript code.

$halt (version 1.7+) MUST be used after done writing javascript section of command to mark the end of the code.

$get(var) (version 1.7+) Used to get variables created by either in javascript block or $set()

$set(var,value) (version 1.7+) Used to set values. Types include numbers, strings, and boolean(true or false).

Disabling Battery Optimizations

Steps

  1. Go to settings
  2. Find battery optimizations
  3. View all apps
  4. Find Bot Commander for Discord
  5. Disable battery optimizations
